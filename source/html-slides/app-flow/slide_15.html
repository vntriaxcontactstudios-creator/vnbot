Selección de Modo — Local vs Servidor

2 MODOS · 4 vs 5 PASOS
// ONBOARDING DECISION
home
USO LOCAL
Para usuarios que quieren control total. Sin dependencias externas. Datos cifrados en dispositivo.
// PIPELINE · 4 NODOS
1 · Crear bóveda local 2 · Elegir LLM local o continuar sin LLM 3 · (Email no obligatorio) 4 · Listo para usar
// VENTAJAS
✓ Sin servidor
✓ Sin email
✓ Funciona offline
// LIMITACIONES
✗ Sin sync entre dispositivos
✗ Solo LLM local o heurística
VS
cloud
CONECTAR A SERVIDOR
Para usuarios con instalación propia o equipo. Sync multi-dispositivo. Workers y colas compartidos.
// PIPELINE · 5 NODOS
1 · Introducir URL 2 · Probar conexión 3 · Login / registro 4 · Seleccionar workspace 5 · Descargar configuración segura
// VENTAJAS
✓ Sync multi-dispositivo
✓ Workers compartidos
✓ Múltiples LLM
// LIMITACIONES
✗ Requiere infraestructura
✗ Servidor puede ver metadatos
CAP. 03 · MODO 15 / 57
- 2 modos mutuamente excluyentes en onboarding. - Local: control total, sin email, sin sync. - Servidor: sync multi-dispositivo, requiere infraestructura. - Email no obligatorio en modo local. - Decisión se toma en paso 2 del wizard.
