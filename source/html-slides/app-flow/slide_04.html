01.1 // PRINCIPIOS

5 Principios del Flujo

5 PRINCIPIOS NO NEGOCIABLES
// FILOSOFÍA OPERACIONAL
01 bolt
CAPTURA RÁPIDA, ACCIÓN EXPLICADA
La captura debe ser inmediata. La ejecución debe ser explícita cuando tenga consecuencias.
// PIPELINE
CAPTURAR ↓ INTERPRETAR ↓ PROPONER ↓ CONFIRMAR ↓ EJECUTAR ↓ INFORMAR
02 visibility
NO FINGIR ÉXITO
Si una operación fue encolada pero no terminó, el estado debe ser En cola o Procesando, no Completado.
// EJEMPLOS
✗ COMPLETADO ← falso éxito ✓ EN COLA ← estado honesto
ESTADO REAL ≠ ÉXITO ASUMIDO
03 restore
RECUPERACIÓN EN CADA ERROR
Cada error debe indicar qué sucedió, qué datos se conservaron, qué no se completó, si se reintentará y qué puede hacer el usuario.
// 5 ELEMENTOS
qué sucedió qué datos se conservaron qué no se completó si se reintentará qué puede hacer el usuario
04 wifi
ONLINE Y OFFLINE EXPLÍCITOS
El usuario debe saber si trabaja local, sin conexión, con sync pendiente, contra servidor o vía proveedor cloud.
// 5 ESTADOS DE RED
LOCAL OFFLINE SYNC_PENDING SERVER CLOUD
05 devices
UN FLUJO, VARIAS INTERFACES
Web, APK, desktop y CLI comparten los mismos estados de dominio, aunque su presentación sea diferente.
