CAMPOS POR OPERACIÓN ENCOLADA:
operation_type: create | update | delete
entity_type: memory | reminder | ...
entity_id: UUID v4
timestamp_local: ISO 8601
version_vector: device + counter
idempotency_key: SHA-256(payload + ts)
PROCESAMIENTO:
check FIFO al reconectar
check Backoff exponencial en fallos
check Máximo 3 intentos antes de conflicto manual
check Conflictos van a vista de resolución

info Las operaciones no se pierden. Se encolan y se reintentan con idempotencia para evitar duplicados.
