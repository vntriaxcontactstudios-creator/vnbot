03.5 // BÓVEDA

Configuración de Bóveda

El usuario debe entender qué protege la passphrase — sin promesas falsas.

5 PUNTOS · 1 ADVERTENCIA
// VAULT CONFIG
// 5 PUNTOS QUE EL USUARIO DEBE ENTENDER
1
¿QUÉ PROTEGE LA PASSPHRASE?
Cifra el contenido de la bóveda: memorias, recordatorios, configuración de agentes, historial de chat. Sin passphrase, los datos son ilegibles.
2
¿QUÉ OCURRE SI SE PIERDE?
No hay recuperación de contenido cifrado sin passphrase. El sistema puede ofrecer restaurar desde exportación previa, pero no descifrar datos sin la clave.
3
¿EXISTE RECUPERACIÓN?
Opcional: backup con clave externa (UX a definir). Recuperación de cuenta sin recuperar bóveda. Restauración desde exportación.
4
¿QUÉ DATOS SE CIFRAN?
Memorias, recordatorios, listas, historial de chat, instrucciones de agentes. NO se cifran: schemas, versiones, timestamps de sync.
5
¿QUÉ PUEDE VER EL SERVIDOR?
Según modo: Local Estricto = nada. Servidor Privado = metadatos. LLM Externo = contexto enviado al proveedor.
warning
NO PROMETER
ZERO-KNOWLEDGE
No se debe presentar una promesa de zero-knowledge si el flujo posterior envía texto a un LLM remoto. La transparencia es obligatoria.
// EJEMPLO CONCRETO
✗ 'Tus datos están protegidos end-to-end'
✓ 'Tu bóveda está cifrada. Cuando usas OpenAI, el contexto enviado se procesa en sus servidores según su política.'
El modo de privacidad se elige explícitamente en onboarding y se puede cambiar en /settings/privacy.
