10.5 // E2E TESTS CHAPTER 10 · CI

Mapa de Pruebas E2E — 10 Tests Críticos

Rutas críticas con cobertura obligatoria en CI.

E2E-01
PRIMER ARRANQUE
Abrir → onboarding → bóveda → primer recordatorio → Hoy
check_circle Usuario llega a Hoy con 1 recordatorio
E2E-02
RECORDATORIO AMBIGUO
Chat → fecha sin hora → aclaración → confirmación → creación
help Sistema pregunta, no inventa
E2E-03
RECORDATORIO RECURRENTE
Chat → recurrencia → ocurrencia → completar una → siguiente permanece
repeat Próxima ocurrencia generada correctamente
E2E-04
FALLO DE LLM
Chat → proveedor caído → heurística → propuesta visible
psychology Fallback funciona, usuario informado
E2E-05
FALLO DE WORKER
Crear recordatorio → worker detenido → estado pendiente → worker recuperado → delivery
build No se pierde, no se marca como completado antes
E2E-06
MEMORIA CONTRADICTORIA
Guardar preferencia → guardar contradicción → comparar → elegir versión
compare_arrows Conflicto visible, usuario decide
E2E-07
OFFLINE
Desconectar → crear memoria → reconectar → sincronizar → resolver conflicto
wifi_off Datos no se pierden, conflicto se resuelve
E2E-08
MCP
Conectar → handshake → scope → tool call → confirmación → auditoría → revocar
extension Tool funciona, permisos respetados, revocable
E2E-09
AUDIO
Permiso → grabar → transcribir → revisar → crear recordatorio
mic Audio capturado, transcript editable, recordatorio creado
E2E-10
ELIMINACIÓN
Exportar → reautenticar → eliminar → revocar → verificar ausencia
delete_forever Cuenta eliminada, integraciones revocadas, datos no accesibles
VNBOT // APP FLOW · CAP. 10 · E2E 56 / 57
- 10 tests E2E cubren rutas críticas del app flow. - Cobertura obligatoria en CI: cada PR debe pasar los 10. - Tipos: arranque, recordatorios (3), fallos (2), memoria, offline, MCP, audio, eliminación. - Criterios verificables: estado visible, no pérdida de datos, permisos respetados. - Eliminación: cuenta + integraciones + datos deben quedar inaccesibles.
