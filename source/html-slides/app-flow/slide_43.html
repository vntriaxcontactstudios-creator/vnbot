MCP — Fallos y Recuperación

Cada fallo tiene manejo explícito. Aislamiento por integración.

link_off HANDSHAKE FALLIDO

Mostrar endpoint, transporte y código de error.

priority_high REGLA CRÍTICA: NO revelar token en el mensaje de error.
→ Acción: Revisar URL/transporte, reintentar con nueva config
ERR: ECONNREFUSED · transport=stdio · endpoint=/usr/local/bin/mcp-server
build_circle TOOL NO DISPONIBLE

Marcar integración degradada. Resto de VNBOT funcional.

Aislamiento: Una tool caída no afecta otras tools del mismo MCP ni otros MCPs
→ Acción: Reintentar con backoff, si persiste pausar tool y notificar
tool.A OK tool.B ✕ tool.C OK — MCP intacto
vpn_key REAUTENTICACIÓN

Token expirado o revocado por el servidor MCP.

check Conservar configuración
check Pausar llamadas pendientes
check Pedir reconectar al usuario
info Nota: No se pierde configuración, solo se re-pide auth.
error RESPUESTA MAL FORMADA

El servidor MCP devolvió JSON inválido o fuera de schema.

priority_high REGLA CRÍTICA: NO ejecutar acciones derivadas de una respuesta inválida.
→ Acción: Registrar error de protocolo, notificar al usuario, pausar integración
