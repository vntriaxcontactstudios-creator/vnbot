// FALLBACK POLICY
01 storage

SQLite / IndexedDB
No se debe fingir que los datos están disponibles. Sin DB no hay app funcional.
// ACCIÓN
Bloquear UI · Mostrar diagnóstico · Ofrecer restauración
02 cloud_off
API REMOTA
Servidor / HTTP
Permitir modo local si existe bóveda local. Operaciones remotas pasan a cola.
// ACCIÓN
OFFLINE + outbox
03 memory
REDIS / COLAS
Background workers
Marcar workers degradados. API disponible, tareas async muestran pendiente.
// ACCIÓN
DEGRADED_WORKER
04 psychology
LLM
Modelo IA
Activar heurística. Captura básica funciona. Sin resúmenes ni inferencias complejas.
// ACCIÓN
DEGRADED_LLM
05 extension
MCP
Integraciones externas
Mostrar integración desconectada. Resto de VNBOT sigue operativo.

MCP_REAUTH_REQUIRED o desconectado
priority_high Sin DB local la app no arranca — es la única dependencia bloqueante. Las demás degradan capacidades sin bloquear el flujo principal.
